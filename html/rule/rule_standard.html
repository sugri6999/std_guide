기본규칙

기본문서

  • 기본문서 코드레이아웃 참고

DOCTYPE HTML5선언

  • HTML5로 선언한다. DTD 선언시 앞에 공백이 오지 않도록 해야하며, 문서 최상단에 선언 한다.
								<!DOCTYPE html>

언어 정의

  • lang="ko"를 기본으로 표기하며 언어에 따라서 해당국가 언어로 표기한다.
  • https://www.w3schools.com/tags/ref_language_codes.asp (참고)
								<html lang="ko"> /* Korean 한국 */
								<html lang="en"> /* English 영어 */
								<html lang="ja"> /* Japanese 일본 */
								<html lang="zh"> /* Chinese 중국 */
								<html lang="ru"> /* Russian 러시아 */
								<html lang="de"> /* German 독일 */

인코딩 정의

  • charset="utf-8"를 기본값으로 사용하고, <head>첫 번째에 선언한다.
								<meta charset="utf-8" />
								<meta charset="euc-kr" />

브라우저 호환성 모드

								<meta http-equiv="X-UA-Compatible" content="IE=edge" />

이미지 툴바 사용 안함

								<meta http-equiv="imagetoolbar" content="no" />

뷰포트 정의

								<meta name="viewport" content="width=device-width,initial-scale=1.0,minimum-scale=1.0,maximum-scale=1.0,user-scalable=no" />

문서정보 정의

  • <meta> : 최소한의 메타정보가 제공되어야 하며 상세내용은 기획과 협의필요.
  • keywords(키워드) : 페이지 키워드를 기재/콤마(,)로 구별하여 기술/동의어와 유의어를 포함.
  • description(페이지 정보 요약) : 구글검색에서, 결과 페이지 내용 미리보기로 사용할 수도 있음.
  • 참고 : 구글 검색엔진 최적화 방법
								<meta name="Author" content="http://www.naver.co.kr" />
								<meta name="title" content="네이버" />
								<meta name="keywords" content="네이버, NHN" />
								<meta name="description" content="네이버, NHN" />

문서제목 표기법

  • <title> : 페이지별 제목이 구분 되어야 한다. (예) 인사말 - 네이버
								<title>페이지명 | 사이트명</title>

표준 문법의 사용

  • HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • DTD를 제외한 모든 엘리먼트와 애트리뷰트는 소문자로 작성한다.
  • 애트리뷰트의 값은 큰따움표("")로 묶는다.
  • 특수기호는 Entity name을 사용하여 entity 코드로 변환한다. Entity 코드는 ISO-8859-1을 기준으로 한다.
  • 마크업은 W3C html5문법에 준하여 사용한다.
  • 종료 태그는 생략하지 않는다.

파일의 인코딩

*.html, *.css, *.js의 파일의 저장 방식을 UTF-8로 설정한다. 반드시 개발과 협의가 필요하다.

문자셋(charset)별 인코딩
문자셋(charset) 기본 인코딩
charset=utf-8 utf-8
charset=euc-kr ANSI

구조, 표현, 동작의 분리

예외사항

								<input type="text" id="formMyClass" class="my-class" style="width:80%;" />
  • table의 col엘리먼트는 style속성으로 적용한다.
  • input[text], select 엘리먼트의 경우 너비의 고정폭 지정 허용한다.
